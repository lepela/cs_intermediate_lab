# Unit 1. 적재 흐름 📥

과정의 **첫 번째 산출물**을 만드는 유닛입니다. 이력서가 첨부된 메일이 도착하면 — 사람이 손대지 않아도 — AI가 핵심 정보를 구조화 추출하고, 요약을 생성하고, SharePoint 리스트에 지원자 항목으로 적재하는 흐름을 처음부터 끝까지 빌드합니다.

```
📧 메일 수신 → 📎 첨부 저장 → 🤖 AI 구조화 추출 → 📋 SP 리스트 적재
```

여기서 만드는 **이력서요약**은 단순한 편의 기능이 아닙니다. 이후 면접관 에이전트가 적합도를 평가할 때 사용하는 **유일한 추론 재료**이고, 그래서 다음 유닛(승인 흐름)에서 사람이 품질을 검증하게 됩니다. "AI 산출물이 하류 작업의 입력이 된다"는 파이프라인 감각을 여기서 익힙니다.

## 이 유닛을 마치면

- 메일 수신을 트리거로 AI 추출→SP 적재까지 자동 실행되는 흐름이 완성됩니다.
- 첨부 파일이 안전한 파일명 규칙으로 문서 라이브러리에 저장됩니다.
- AI 프롬프트 액션의 구조화 출력(JSON)을 흐름에서 다루는 방법을 익힙니다.

{: .warning }
이력서는 **PDF 전용**입니다. DOCX·HWP는 AI 프롬프트 액션이 Bad Request로 거부합니다.
