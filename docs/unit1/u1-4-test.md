## 🎯 학습 목표

- 완성한 적재 흐름을 **실제 메일로 실행**해 동작을 확인할 수 있다.
- 적재 결과(항목·요약·링크)를 **목록에서 검증**할 수 있다.
- 적재 직후 항목이 **보류 중** 상태인 이유를 이해하고 다음 유닛으로 연결한다.

## ⏱ 예상 소요 시간

| 구분 | 시간 |
|---|---|
| 강사 설명 | 2분 |
| 실습 | 6분 |
| 버퍼 | 1분 |

---

## 준비물

- 1-3까지 완료한 **적재 흐름**(저장됨)
- **이력서 PDF 6개**가 첨부된 테스트 메일 (실습용 샘플)

---

## 단계별 가이드

### 1단계. 흐름 저장 및 켜기

적재 흐름을 **저장**하고, 상태가 **켜짐(On)** 인지 확인합니다. 자동 트리거 흐름은 켜져 있어야 메일을 감지합니다.

<!-- SCREENSHOT: u1-s09 — 적재 흐름 저장·켜짐 상태 -->

### 2단계. 테스트 메일 발송

이력서 PDF 6개를 첨부한 메일을, **Outlook 규칙이 인식하는 제목**(예: 제목에 `[이력서샘플]` 포함)으로 보냅니다. 규칙이 이 메일을 채용 전용 폴더로 옮기고, 그 폴더를 감시하는 트리거가 실행됩니다.

{: .note }
첨부를 한 통에 6개 넣어 보냅니다. Apply to each가 6번 돌며 각 PDF를 추출·적재합니다.

{: .warning }
제목에 규칙 키워드가 없으면 메일이 전용 폴더로 분류되지 않아 **흐름이 아예 깨어나지 않습니다**. 트리거가 안 돌 때 가장 먼저 메일이 전용 폴더에 들어왔는지 확인하세요.

### 3단계. 실행 기록 확인

흐름의 **28일 실행 기록**에서 방금 실행이 **성공**으로 표시되는지 확인합니다. 실패 시 어느 액션에서 멈췄는지 펼쳐서 봅니다.

<!-- SCREENSHOT: u1-s10 — 흐름 실행 기록, 성공 표시 -->

### 4단계. 적재 결과 검증

지원자 마스터 목록을 열어 아래를 확인합니다.

- 새 항목 **6개**가 추가됐는가
- 제목이 `[날짜]이름_직군` 형식인가
- **이력서요약**이 채워졌고 HTML 태그가 섞이지 않았는가
- **이력서링크**를 클릭하면 보관함의 PDF가 열리는가

<!-- SCREENSHOT: u1-s11 — 지원자 마스터에 적재된 6개 항목 -->

---

## ✅ 체크포인트

- [ ] 적재 흐름이 **성공**으로 실행됐습니다.
- [ ] 지원자 마스터에 지원자 **6명**이 추가됐습니다.
- [ ] 이력서요약·이력서링크가 정상이고, 링크로 원본 PDF가 열립니다.
- [ ] 추가된 항목들이 **보류 중** 상태로 보입니다.

---

## 핵심 정리

| 항목 | 내용 |
|---|---|
| 실행 기록 | 자동 흐름의 디버깅 출발점 — 성공/실패와 멈춘 액션 확인. |
| 적재 검증 | 항목 수·요약·링크 3가지를 눈으로 확인. |
| 보류 중 | 콘텐츠 승인이 켜진 목록이라 적재 직후 자동 보류 → 다음 유닛에서 승인. |

{: .important }
적재된 지원자가 곧바로 면접관 에이전트에 보이지 않는 것은 **정상**입니다. Unit 0에서 켠 콘텐츠 승인 때문에 새 항목이 **보류 중**으로 들어가고, 사람이 요약 품질을 검증해 승인해야 비로소 하류로 흘러갑니다. 그 승인 자동화가 다음 유닛입니다.
