## 개념

5-2에서 토픽에 게이트를 넣었습니다. 그런데 "대상이 한 명인지"는 **어떻게 보장**할까요? 이름은 유일하지 않습니다.

- "김지훈"이 **두 명**이면 → 누구를 바꿔야 하나?
- "김지후"라고 잘못 부르면 → **0명**이 나오는데, 토픽이 그냥 통과시키면?

이 유닛의 불변식은 **"대상을 정확히 한 명으로 특정한다"**입니다. 그리고 그걸 지침(소프트)이 아니라 **토픽의 첫 번째 게이트에 보이게**(하드) 박는 것이 핵심입니다 — 조회 결과의 **행 갯수**로 분기하면 됩니다.

```
토픽 진입
  → [게이트 ①] 이름으로 항목 조회 → count 확인
       0건   → "해당 이름의 지원자가 없습니다" → 토픽 종료 (갱신 X)
       2건↑  → "동명이인 N명, 이메일이나 ID로 특정해 달라" → 토픽 종료 (갱신 X)
       1건   → 게이트 ②③ 통과 → 카드 → 흐름 호출
```

이렇게 하면 **"하나일 때만 바꾼다"**가 토픽 구조 자체로 보장됩니다. 모델이 잘 골라주길 기대하는 게 아니라, 시스템이 강제합니다.

---

## 단계별 가이드

### 1단계. 토픽 게이트 ① — 행 갯수 분기

토픽 진입 직후, 조회 결과 배열의 **길이**로 조건을 나눕니다.

```
length( 항목_가져오기의 결과 값 )
```

| 행 갯수 | 처리 |
|---|---|
| 0 | "그 이름의 지원자가 없습니다" → 모든 토픽 종료 |
| 2 이상 | "동명이인이 N명입니다. 이메일/ID로 특정해 주세요" → 모든 토픽 종료 |
| 1 | 게이트 ②③으로 통과 |

<!-- SCREENSHOT: u5-s08 — 토픽 게이트 ① 행 갯수 3분기(0 / 1 / 2+) -->

{: .note }
가장 간단한 버전은 **단일 조건 `행 갯수 = 1`** 하나입니다 — 참이면 다음 게이트, 거짓이면 "유일하게 특정되지 않아 변경하지 않음"(0건·다건을 한 메시지로). 분기를 더 줄이고 싶을 때 이 형태를 씁니다.

### 2단계. ID로 단건 갱신 — 흐름이 마무리

게이트를 모두 통과하고 사람이 카드에서 확정을 누르면, **상태변경 흐름**이 호출됩니다. 흐름은 **그 행의 ID**를 업데이트 키로 씁니다. 이름이 아니라 ID로 바꾸므로 "혹시 다른 사람"이 끼어들 여지가 없습니다.

```
흐름: 항목 업데이트 → ID = 조회 결과 첫 행의 ID (First(.value)?['ID'])
```

<!-- SCREENSHOT: u5-s09 — 흐름 내 ID 기준 단건 업데이트 -->

{: .important }
토픽이 **이름 → 조회 → 유일성 검증 → 카드 확인**을 끝내고, 흐름이 **ID로 단건 갱신**을 마무리합니다. 에이전트가 "대충 김지훈"이라고만 넘겨도, 둘이면 토픽이 막고, 하나일 때만 그 ID로 정확히 바꿉니다. 이것이 "대상 특정을 시스템에 가둔다"는 의미입니다.

### 3단계. 필수 입력 stall 풀기 — AI적합도를 선택적으로

5-2의 입력 중 **AI적합도가 '필수'**면, 에이전트가 그 값을 못 채울 때 흐름이 **항목 업데이트 직전에서 멈춥니다**(stall). 설명만으론 "필수"를 넘길 수 없습니다.

트리거에서 **AI적합도를 '선택적(optional)'으로** 바꾸거나 기본값을 주고, 흐름을 **게시**한 뒤 다시 테스트합니다.

<!-- SCREENSHOT: u5-s10 — AI적합도 입력을 선택적으로 변경 -->

{: .warning }
트리거의 입력 스키마를 바꾸면 **에이전트 쪽 도구가 입력을 재동기화**해야 할 수 있습니다(도구 새로고침/재첨부). 게시 후 첫 호출에서 입력 칸이 그대로인지 확인하세요. 또 운영 리스트의 내부 컬럼명은 교안 표시명과 다를 수 있으니(예: `AI적합도` = `AI_Adaptation`), 필터·식에는 **운영 리스트 기준 내부명**을 씁니다.

---

## ✅ 체크포인트

- [ ] 조회 직후 **행 갯수**로 0 / 1 / 2+ 를 나눕니다.
- [ ] 1건일 때만 **그 행의 ID**로 단건 갱신합니다.
- [ ] 0건·동명이인은 갱신하지 않고 사유를 돌려줍니다.
- [ ] AI적합도 stall을 풀고 흐름을 **게시**했습니다.

---

## 핵심 정리

| 항목 | 내용 |
|---|---|
| 토픽 게이트 ① | 0건=없음 / 2+건=동명이인 → 토픽 종료. 1건만 통과. |
| ID 단건 갱신 | 토픽이 이름→ID 해소, 흐름이 ID로 단건 갱신. |
| 구조적 방어 | "하나일 때만"이 지침이 아니라 토픽 노드에 박힘. |
| stall 해소 | 필수 입력을 선택적으로 → 게시 → 재동기화 확인. |
