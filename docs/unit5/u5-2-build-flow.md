## 🎯 학습 목표

- 검토상태를 바꾸는 **면접 확정 흐름**을 만들고 에이전트의 **도구로 붙인다**.
- **전제조건 게이트**(승인됨 + 적합도 평가됨)를 흐름 안에 결정적으로 넣는다.
- 서로 종속된 **여러 컬럼을 한 번에(원자적으로)** 갱신한다.

## 개념

흐름도 결국 **도구**입니다. 그래서 구조는 Unit 4와 같습니다 — 에이전트가 지침에 따라 이 도구를 부르고, **도구의 속이 결정적 흐름**일 뿐입니다.

```
"1차 합격 처리해줘"
  → (지침) '검토상태 변경' 도구를 호출
      → [흐름 속] ① 전제조건 검사 → ② 다중 컬럼 원자 갱신 → ③ 결과 반환
```

핵심은 **흐름 안에 불변식을 가둔다**는 것입니다. "승인도 안 된 사람을 1차합격시키지 않는다", "검토상태와 면접확인필요를 함께 맞춘다" 같은 규칙을 확률적 LLM이 아니라 **흐름이 결정적으로** 지킵니다.

---

## 단계별 가이드

### 1단계. 흐름 만들기 + 입력 정의

CS에서 새 흐름 **`검토상태 변경`** 을 만들고, 트리거를 **"에이전트가 흐름을 호출할 때"** 로 둡니다. 입력은 두 가지입니다.

| 입력 | 설명 |
|---|---|
| 지원자 ID(또는 이름) | 대상 지원자 |
| 목표 검토상태 | 1차합격 / 보류 / 불합격 등 |

<!-- SCREENSHOT: u5-s03 — 흐름 트리거 + 입력 정의 -->

### 2단계. 전제조건 게이트 — 자격 검사

본 갱신 **전에** 조건을 검사합니다. 통과 못 하면 갱신하지 않고 사유를 돌려줍니다.

| 전제조건 | 통과 못 하면 |
|---|---|
| 승인상태 = 승인됨 | "AI 요약 승인이 끝나지 않은 지원자" → 차단 |
| AI적합도 ≠ 미적용 | "적합도 평가 전이라 1차합격 불가" → 차단 |

<!-- SCREENSHOT: u5-s04 — 전제조건 조건 분기 -->

{: .important }
이 게이트가 흐름의 핵심 가치입니다. 커넥터+지침이라면 "확률적으로 대개 지킴"이지만, 흐름은 **조건을 못 넘으면 절대 갱신이 일어나지 않습니다**(결정적). "1차합격은 평가가 끝난 승인된 사람만"이라는 불변식을 시스템이 강제합니다.

### 3단계. 다중 컬럼 원자 갱신

전제조건을 통과하면 **항목 업데이트**로 종속 컬럼을 함께 맞춥니다 — 한 번의 갱신으로 묶어 중간에 어긋나지 않게 합니다.

| 컬럼 | 갱신 |
|---|---|
| 검토상태 | 목표 값(예: 1차합격) |
| 면접확인필요 | 상태에 맞게 정리(예: 확정 시 해제) |

<!-- SCREENSHOT: u5-s05 — 다중 컬럼 업데이트 -->

{: .note }
컬럼들이 **서로 종속**돼 있어 따로따로 바꾸면 데이터가 어긋납니다. 흐름은 이들을 **한 묶음**으로 갱신해 "전부 바뀌거나 전부 안 바뀌거나"를 보장합니다 — 이게 트랜잭션의 원자성입니다.

### 4단계. 흐름을 도구로 붙이기

이 흐름을 면접관 에이전트의 **도구**로 추가하고, 지침에서 "검토상태 변경은 이 도구로만"이라고 못박습니다(다음 서브유닛).

<!-- SCREENSHOT: u5-s06 — 흐름을 에이전트 도구로 추가 -->

{: .warning }
검토상태를 바꾸면 콘텐츠 승인이 켜진 리스트에서 **승인상태가 보류중으로 리셋**될 수 있습니다(5-1의 서버 부작용). 운영에서는 흐름 안에서 승인상태를 다시 정리해 그 지원자가 조회에서 사라지지 않게 합니다 — 지침으론 못 하고 **흐름 안에서만** 처리할 수 있는 일입니다.

---

## ✅ 체크포인트

- [ ] **검토상태 변경** 흐름이 만들어져 있습니다.
- [ ] 흐름이 **전제조건**(승인됨 + 적합도 평가됨)을 검사합니다.
- [ ] 통과 시 **여러 컬럼을 한 번에** 갱신합니다.
- [ ] 흐름이 에이전트의 **도구**로 붙어 있습니다.

---

## 핵심 정리

| 항목 | 내용 |
|---|---|
| 흐름 = 도구의 속 | 호출은 도구+지침, 결정적 처리는 흐름 안. |
| 전제조건 게이트 | 승인됨 + 적합도 평가됨 아니면 절대 갱신 안 함. |
| 원자 갱신 | 종속 컬럼을 한 묶음으로 — 전부 아니면 전무. |
| 서버 부작용 | moderation 리셋 정리도 흐름 안에서만 가능. |
