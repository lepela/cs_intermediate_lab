## 🎯 학습 목표

- 면접관 에이전트에 **SharePoint 커넥터 도구**("항목 가져오기")를 연결한다.
- 도구 출력이 **자동으로 답변 근거**가 되고, 어떤 도구를 쓸지는 **지침으로 못박아** 오답 범위를 줄이는 원리를 이해한다.
- **하나의 도구**가 지침에 따라 세 기능(C1·C2·C3)으로 갈라지는 그림을 잡는다.

## 개념

Unit 3의 마지막에서 에이전트는 "김지훈 상태?"에 답하지 못했습니다. **실시간 데이터에 닿는 도구가 없었기 때문**입니다. 이번 유닛에서 그 도구를 딱 하나 붙입니다 — SharePoint **항목 가져오기(Get items)** 커넥터.

{: .important }
이 유닛의 핵심 주장은 **"세 기능 = 세 흐름이 아니다"** 입니다. 조회(C1)·평가(C2)·질문(C3)은 서로 다른 흐름을 만드는 게 아니라, **같은 커넥터 도구 하나**를 에이전트가 지침에 따라 다르게 활용하는 것입니다. 모델이 좋아질수록 "조회·추론"은 생성형 오케스트레이션이 흡수합니다.

| | 옛 방식(흐름 시대) | 이 과정(커넥터+지침) |
|---|---|---|
| 조회 | 흐름 1개 | 도구 1개 + 지침 |
| 평가 | 흐름 1개 | 같은 도구 + Knowledge + 지침 |
| 질문 | 흐름 1개 | 같은 도구(+파일 fetch) + 지침 |
| 빌드량 | 큼 | 작음 (대신 테스트·지침이 일) |

---

## 단계별 가이드

### 1단계. 도구 추가 — SharePoint 항목 가져오기

면접관 에이전트의 **도구(Tools)** 탭에서 **`+ 도구 추가`** → 커넥터 **SharePoint — 항목 가져오기**를 선택합니다.

### 2단계. 대상 목록 지정

| 항목 | 값 |
|---|---|
| 사이트 주소 | HR Lab |
| 목록 이름 | 지원자 마스터 |

<!-- SCREENSHOT: u4-s02 — 항목 가져오기, 지원자 마스터 지정 -->

{: .note }
연결을 처음 쓰면 SharePoint **연결 동의** 카드가 뜹니다. 허용해야 도구가 데이터를 읽습니다. 동의 주체(작성자 연결 vs 최종 사용자)는 운영 정책에 따라 정하지만, 실습에서는 기본 연결로 진행합니다.

### 3단계. 도구 설명 달기 — 오케스트레이터가 언제 쓸지 알도록

에이전트(오케스트레이터)는 사용자의 말과 도구 **설명**을 보고 이 도구를 호출할지 판단합니다. 그래서 설명을 명확히 적습니다 — 예: `SharePoint 지원자 목록에서 항목을 가져옵니다`. 도구가 돌려준 항목은 오케스트레이터가 **자동으로 답변 근거로 활용**하므로, 출력을 근거 자리에 따로 배선할 일은 없습니다.

<!-- SCREENSHOT: u4-s03 — 도구 세부 정보(이름·설명·입력) -->

{: .note }
흐름(디자이너)으로 만들 때는 도구 출력을 생성형 답변 노드의 근거 자리에 직접 배선해야 했습니다. 하지만 **커넥터를 도구로 붙이면 오케스트레이터가 근거화를 대신**합니다 — 커넥터 방식이 더 간단한 이유 중 하나입니다.

### 4단계. 지침으로 도구 사용을 못박기 — 오답 범위 줄이기

도구 설명만으로도 대개 맞는 도구를 찾습니다. 하지만 **라우팅을 운에 맡기지 않습니다.** 지침에 "지원자 정보는 언제나 이 도구로 가져온다"를 명시해, 엉뚱한 경로로 답할 가능성을 줄입니다.

```
[경계 — 도구 사용]
- 지원자 정보를 가져올 때는 언제나 '항목 가져오기' 도구를 사용한다.
- 도구가 정보를 주지 않으면 지어내지 말고 '확인 필요'로 둔다.
```

<!-- SCREENSHOT: u4-s03b — 지침: 도구 사용 못박기(경계) -->

{: .important }
**명시 > 자동.** 에이전트가 "알아서 잘하겠지"에 기대면 답이 맞을 때도, 틀릴 때도 있습니다(비결정적). 줄일 수 있는 모호함은 줄이는 게 원칙 — 지침으로 도구를 못박으면 **오답이 나올 수 있는 범위 자체가 좁아집니다.** 이렇게 지침으로 에이전트의 행동 범위를 좁히는 제약을 **가드레일(guardrail)** 이라고 합니다 — 도로 난간처럼 에이전트가 길 밖으로 못 나가게 막는 것이죠. 이번 유닛 내내 우리는 가드레일을 한 줄씩 세워 갑니다.

## ✅ 체크포인트

- [ ] 에이전트에 **항목 가져오기** 도구가 추가돼 있습니다.
- [ ] 도구가 **지원자 마스터** 목록을 대상으로 합니다.
- [ ] SharePoint 연결이 동의·허용돼 있습니다.
- [ ] 도구에 명확한 **설명**이 달려 있습니다.
- [ ] 지침에 "지원자 정보는 이 도구로 가져온다"가 **명시**돼 있습니다.

## 핵심 정리

| 항목 | 내용 |
|---|---|
| 도구 1개 | 항목 가져오기 하나로 C1·C2·C3 전부 커버. |
| 근거화 = 자동 | 도구 출력은 오케스트레이터가 알아서 근거로 씀(수동 배선 불필요). |
| 라우팅 = 명시 | 어떤 도구를 쓸지는 지침에 못박아 오답 범위를 줄인다. |
| 패러다임 | 흐름을 줄이고 지침·테스트로 기능을 가른다. |
