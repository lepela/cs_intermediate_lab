## 🎯 학습 목표

- 면접 질문도 **기본은 요약**으로 만들고, 더 깊은 질문은 **원문을 1건 fetch**해 보강한다.

- 원문 접근 권한은 **에이전트가 아니라 연결 신원·SharePoint 권한**이 정한다는 것을 안다.

## 개념

면접 질문 생성(C3)도 **기본은 이력서요약**으로 만듭니다. 다만 면접 질문은 깊을수록 좋으니, 사용자가 원하거나 더 정밀한 질문이 필요하면 **원본 이력서 1건**을 가져와 보강합니다 — 4-3에서 정한 **2단 구조**(요약 기본 / 원문 에스컬레이션)를 그대로 따릅니다.

{: .important }
**원본 이력서는 Knowledge에 넣지 않습니다.** 대신 그 지원자 항목의 **이력서링크로 그 1건만** 가져옵니다(fetch-by-link). 수백 명의 이력서를 RAG에 올리면 검색이 **올려둔 이력서 전체 더미(코퍼스)** 에서 일어나 **다른 지원자 내용이 섞이지만**, 링크로 1건만 가져오면 지원자 수가 아무리 많아도 안전합니다. **전체에서 뒤지는 검색이 아니라, 링크로 그 한 명만 콕 집어 가져오는 것**이죠.

---

## 단계별 가이드

### 1단계. 원문 가져오기 수단 연결 — 경로사용 파일콘텐츠

원문은 SharePoint **`경로로 파일 콘텐츠 가져오기`(Get file content using path)** 도구로 가져옵니다. **사이트는 고정**하고, 어떤 파일을 가져올지 **파일 경로는 에이전트(AI)가 이력서링크에서 채우게** 합니다.

<!-- SCREENSHOT: u4-s10 — 경로사용 파일콘텐츠 도구(사이트 고정 / 경로는 AI) -->

{: .note }
C1·C2는 목록(요약·필드)만으로 됐지만, 원문은 파일이라 가져오는 수단이 하나 더 붙습니다. 그래도 "여러 흐름"이 아니라, 한 지원자를 깊이 보기 위한 **1건 fetch**입니다. (커넥터 단독으로 PDF 내용까지 읽혀, "읽기=커넥터"가 여기서도 유지됩니다.)

{: .warning }
**원문 가져오기는 로딩이 꽤 깁니다.** 그래서 매번이 아니라 **요청·심층 시에만** 부릅니다(2단 구조). 라이브 시연 때는 응답이 늦을 수 있음을 미리 알려 두는 게 좋습니다.

### 2단계. 질문 생성 지침 추가 — 요약 기본, 원문 에스컬레이션

```
[면접 질문 생성]
- 기본은 이력서요약을 근거로 질문을 만든다.
- 사용자가 "원문 / 정밀 / 더 깊은 질문"을 요청하면, 이력서링크로
  '경로사용 파일콘텐츠' 도구를 써서 원본 1건을 가져와 보강한다.
- 평가 루브릭의 약한 축을 확인할 질문을 우선 만든다.
- 원문에 없는 경력·성과를 지어내지 않는다.
```

<!-- SCREENSHOT: u4-s11 — 지침: 면접 질문 생성(2단 구조) -->

### 3단계. 테스트 — 요약 기반 → 원문 보강

먼저 "김지훈 면접 질문 만들어줘"로 **요약 기반** 질문이 나오는지 봅니다. 이어서 "원문까지 보고 더 깊게"로 **원문을 가져와** 구체적 수치·경험을 짚는 질문으로 보강되는지 확인합니다.

<!-- SCREENSHOT: u4-s12 — 요약 기반 질문 → 원문 보강 질문 -->

{: .warning }
**원문 접근 권한은 에이전트가 막는 게 아닙니다.** 누가 이력서 파일을 읽을 수 있는지는 두 가지가 정합니다 — ① 커넥터의 **연결 신원**(작성자 연결이면 만든 사람 권한으로, 최종 사용자 인증이면 대화하는 사람 권한으로 읽음), ② **SharePoint 문서 라이브러리 자체의 권한**. 이 둘이 진짜(하드) 경계입니다. 지침·토픽은 "이 기능을 제공할지" 정도만 정할 뿐(소프트)이라, 보안 경계가 될 수 없습니다.

{: .note }
특히 **작성자 연결**을 쓰면 대화하는 모든 사용자가 "만든 사람이 볼 수 있는" 파일까지 받을 수 있습니다 — 의도와 다르면 위험합니다. 사용자별로 제한하려면 **최종 사용자 인증 + SharePoint 권한**으로 가야 합니다. "지침에 보여주지 마"라고 적는 건 우회 가능한 소프트 통제라 보안이 되지 못합니다.

## ✅ 체크포인트

- [ ] 기본 면접 질문이 **요약 기반**으로 생성됩니다.
- [ ] 원문 요청 시 **경로사용 파일콘텐츠**로 원본 1건을 가져옵니다.
- [ ] 원문 보강 질문이 구체적 수치·경험을 짚습니다.
- [ ] 원문에 없는 내용을 지어내지 않습니다.

## 핵심 정리

| 항목 | 내용 |
|---|---|
| C3 = 1건 fetch | 원문은 Knowledge ❌, 이력서링크로 그 1건만(경로사용 파일콘텐츠). |
| 기본 요약 / 원문 에스컬레이션 | 질문도 기본은 요약, 원문은 요청·심층 시. 원문 fetch는 느림. |
| 교차 오염 회피 | 1건 fetch라 지원자 수와 무관하게 안전. |
| 권한 경계 = 다른 위치 | 연결 신원 + SharePoint 권한이 막는다. 지침은 보안 경계 아님(소프트). |
