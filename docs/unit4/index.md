# Unit 4. 모듈 1 — 조회·평가·질문 (커넥터+지침) 🔍

흐름을 하나도 만들지 않고 어디까지 갈 수 있을까요? 이 유닛에서는 SharePoint **커넥터 도구 두 개**("항목 가져오기"·"경로로 파일 콘텐츠 가져오기")와 Knowledge, 지침만으로 세 가지 **기능**을 완성합니다 — **C1 지원자 조회, C2 루브릭 기반 적합도 평가, C3 면접 질문 생성**. 셋은 별개의 흐름이 아니라 **두 커넥터 도구를 지침으로 조율하는 세 가지 의도**입니다. C1·C2는 목록 항목을 읽는 "항목 가져오기" 하나로 충분하고, C3에서 이력서 원문이 필요할 때 "경로로 파일 콘텐츠 가져오기"가 추가됩니다.

빌드보다 **테스트가 많은** 유닛입니다. 프롬프트를 바꿔 가며 에이전트의 행동을 관찰하고, 지침 한 줄이 결과를 어떻게 바꾸는지 실험합니다. "조회와 추론은 생성형 오케스트레이션이 흡수한다"는 이 과정의 절반짜리 결론을 여기서 직접 확인합니다.

## 이 유닛을 마치면

- 승인된 지원자만 조회하는 게이트가 **지침만으로** 동작합니다.
- 이력서 요약 ↔ 평가 루브릭 대조로 적합도 등급이 산출됩니다.
- 견고함이 도구가 아니라 **지침에서 나온다**는 것을, 그리고 그 쉬운 길의 한계와 더 단단한 길(흐름)의 차이를 체득합니다.

{: .note }
여기서 만드는 것은 흐름이 아니라 **도구 + 지침**입니다. 같은 기능을 흐름으로 만들던 시대와 비교하면 빌드 양이 극적으로 줄어듭니다 — 대신 테스트와 지침 다듬기가 일이 됩니다.
