## 개념

적합도 평가(C2)는 **새 흐름이 아닙니다.** 조회로 가져온 지원자의 **이력서요약**(SharePoint)과 **평가 루브릭**(Knowledge)을 에이전트가 대조해 등급을 내는 것입니다. 같은 도구 + Knowledge + 지침의 조합입니다.

루브릭은 5개 평가 항목을 보고, 항목별 점수를 더하는 방식이 아니라 **전체적으로 종합해 등급을 판단**합니다.

| 평가 항목 | 살피는 내용 |
|---|---|
| R1 직군 요건 〔게이트〕 | 필수 요건 충족 여부 (JD) |
| R2 경력 레벨 | 연차·레벨 적합성 |
| R3 성과·임팩트 | 정량 성과 신호 |
| R4 협업·소통 | 팀워크 근거 |
| R5 인재상 부합 | 회사 인재상 4축 |

{: .note }
R1은 **게이트 항목**입니다. 필수 요건을 충족하지 못하면 나머지 항목이 좋아도 등급이 올라가지 않습니다. 전체 등급은 "대부분 충족 → 적극 추천"처럼 전체 인상을 종합해 결정합니다. 항목별 점수를 합산하지 않는 이유는 LLM이 산술 일관성이 낮아 오히려 오차가 생기기 때문입니다.

## 단계별 가이드

### 1단계. 평가 지침 추가

지침에 평가의 입력·기준·출력 형식을 명시합니다.

```
[적합도 평가]
- 평가 대상의 근거는 지원자의 "이력서요약"(SharePoint)이다.
- 채점 기준은 "평가 루브릭"(Knowledge)을 따른다 — R1~R5 5축.
- R1(직군 요건)은 게이트다: 필수 요건 미충족이면 상위 등급을 주지 않는다.
- 결과 산출 근거는 등급 + 축별 근거(강/중/약)로 정성적으로 제시한다.
- 적합도평가 결과는 적극추천/추천/면접확인필요/부적합 중 하나로 제시한다. 기본값은 미적용이다.
```

{: .note }
지침의 "축"과 "축별 근거"는 위 표의 평가 항목(R1~R5)을 가리키는 루브릭 원문 표기입니다.

{: .warning }
**등급 어휘를 못박는 이유:** 마지막 줄이 없으면 모델이 "상위 등급"처럼 루브릭에 없는 등급명을 즉흥으로 만들 수 있습니다(GPT-4.1에서 관측됨). 허용 등급 4종(적극추천/추천/면접확인필요/부적합)과 기본값(미적용)을 지침에 명시하면 출력이 루브릭 어휘로 고정됩니다 — "줄일 수 있는 모호함은 줄인다"는 가드레일 원칙의 또 다른 예입니다.

### 2단계. 테스트 — 특정 지원자 평가

"김지훈 적합도 평가해줘"처럼 물어, 에이전트가 요약↔루브릭을 대조해 **등급 + 축별 근거**를 내는지 확인합니다.

<!-- SCREENSHOT: u4-s09 — 적합도 평가 결과(등급 + 축별 근거) -->

### 3단계. 한계 관찰 — 요약에 수치가 없으면 약해진다

"**오다은** 적합도 평가해줘"라고 물어봅니다.

<!-- SCREENSHOT: u4-s09b — 오다은 적합도 평가 결과(R3·R4·R5 중) -->

종합 등급은 **"추천"** 이 나옵니다. 그런데 축별 근거를 보면 R3 성과·임팩트, R4 협업·소통, R5 인재상 부합이 모두 **"중"** 이고, 근거마다 "실질적 임팩트 확인 제한", "팀 내 소통 사례 명시 없음" 같은 코멘트가 붙어 있습니다.

{: .warning }
**"추천"이 나와도 안심할 수 없습니다.** R1·R2가 강하면 R3~R5가 중이어도 추천이 나와요. 문제는 이 추천이 맞는 추천인지 알 수 없다는 겁니다 — 요약에 수치와 사례가 충분히 담겼다면 R3~R5도 강이 나왔을 수 있고, 등급은 "적극추천"이었을 수도 있어요. 하지만 **등급만 보면 그 차이가 전혀 보이지 않습니다.**

{: .important }
**등급이 지원자 실력이 아니라 "요약에 수치와 사례가 살아남았는지"를 추적하는 셈입니다.** 이 문제를 지금 이 단계에서 발견하면 — 되돌릴 방법이 없습니다. 요약을 다시 생성하려면 적재 흐름(Unit 1)부터 다시 돌려야 해요. **Unit 2의 승인 HITL은 바로 이 순간을 막기 위한 사전 방어**였습니다 — C2의 평가 재료가 곧 그 요약이니까요.

## ✅ 체크포인트

- [ ] 특정 지원자에 대해 **등급 + 축별 근거**가 나옵니다.

## 핵심 정리

| 항목 | 내용 |
|---|---|
| C2 = 대조 | 이력서요약(SP) ↔ 루브릭(Knowledge). 새 흐름 아님. |
| 5개 평가 항목 | R1 게이트 + R2~R5, 점수 합산 없이 전체 인상으로 종합 판단. |
| 요약 의존 | 평가 품질 = 요약 품질. 수치·사례 누락 시 R3~R5 약화, 등급에서 티 안 남. |
| 파이프라인 | Unit 1 수치 보존 → Unit 2 품질 검증(사전 방어) → C2 평가로 연결. |
