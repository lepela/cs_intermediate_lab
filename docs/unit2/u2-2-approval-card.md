## 개념

승인 카드의 **세부 정보(Details)** 칸은 담당자가 Teams·Outlook에서 보게 될 본문입니다. 여기에 "이 지원자를 승인할지 판단할 재료" — 이름·경력·직군·이력서 요약·원본 링크 — 를 싣습니다.

문제는 이 칸의 렌더러가 **Markdown만** 해석하고, 그것도 까다롭다는 점입니다. HTML을 넣으면 태그가 날 것으로 노출되고, 빈 줄을 빼먹으면 서식이 통째로 무너집니다. 그래서 **규칙을 지킨 Markdown**으로 써야 합니다.

{: .important }
이것이 비동기 HITL의 특징입니다 — **카드 UI를 우리가 직접 통제할 수 없습니다.** 버튼 라벨·레이아웃은 못 바꾸고, 본문 서식만 Markdown 규칙 안에서 다룰 수 있습니다. (반대로 대화형 HITL은 챗봇 대화라 지침으로 더 유연하게 통제합니다 — Unit 5에서 대비됩니다.)

## 단계별 가이드

### 1단계. 세부 정보에 카드 본문 작성

**승인 시작 및 대기**의 **세부 정보** 칸에 아래 구조로 작성합니다. `{ }`는 동적 콘텐츠 칩으로 채웁니다.

```markdown
### 지원자 승인 요청

- **이름**: {지원자이름}
- **경력**: {경력사항}
- **직군**: {지원직군}
- **이력서링크**: [이력서 보기]({이력서링크})

**이력서 요약**

{이력서요약}

※ 거부 시 보완 요청 사유를 주석에 남겨 주세요. 주석 내용은 지원자 안내 메일에 그대로 사용됩니다.
```

![세부 정보 Markdown 카드 완성]({{ site.baseurl }}/assets/unit2/u2-s03.png)  
*이름·경력·직군·이력서링크·이력서요약·주의사항 순서로 작성 완료. 각 칩은 동적 콘텐츠 패널에서 삽입합니다.*

### 2단계. Markdown 5규칙 점검

작성한 본문이 아래 규칙을 지키는지 확인합니다.

| # | 규칙 | 어기면 |
|---|---|---|
| 1 | HTML 태그 금지. 링크는 `[텍스트](URL)`만 | `<a href>`가 날 텍스트로 노출 |
| 2 | URL에 공백 없어야 함 | 공백에서 링크가 잘려 깨짐 |
| 3 | 블록 사이에 **빈 줄** 필수 (단일 줄바꿈 무시) | 본문이 헤더에 빨려 들어가 전체가 크고 굵게 |
| 4 | `**볼드**` 별표 안쪽 공백 금지 (`** 이름 **` ✗) | 볼드가 안 먹고 별표 노출 |
| 5 | 줄 나열은 불릿(`- `)이 가장 안전 | 줄바꿈이 뭉개짐 |

{: .warning }
**이력서링크가 깨진다면** 원본 파일명에 공백이 있는지 의심하세요(규칙 2). Unit 1에서 파일명을 `{시각}_{GUID8}.pdf`로 만든 이유가 바로 이 카드 링크까지 보호하기 위해서입니다.

{: .warning }
**요약에 `<div class="ExternalClass...">` 같은 태그가 섞여 나온다면**, 이력서요약 컬럼이 "서식 있는 텍스트"로 되어 있을 가능성이 큽니다. Unit 0에서 **일반 텍스트**로 만들었는지 확인하세요(규칙 1과 같은 뿌리).

{: .note }
세부 정보를 편집할 때 동적 콘텐츠 칩 주변에 따옴표 잔재(`("URL")`)가 남기 쉽습니다. 렌더링 화면만으로는 판별이 어려우니, **코드 보기로 실제 정의 문자열을 확인**하며 수정하는 것이 안전합니다.

## ✅ 체크포인트

- [ ] 세부 정보에 이름·경력·직군·이력서링크·이력서요약이 불릿 구조로 들어 있습니다.
- [ ] 이력서링크가 `[이력서 보기](URL)` Markdown 문법으로 작성되어 있습니다.
- [ ] 블록 사이에 빈 줄이 있고, 볼드 별표 안쪽에 공백이 없습니다.
- [ ] (테스트 시) 카드에서 요약이 순수 텍스트로, 링크가 클릭 가능하게 보입니다.

---

## 핵심 정리

| 항목 | 내용 |
|---|---|
| 카드 = Markdown 전용 | HTML 미해석. 링크는 `[텍스트](URL)`만. |
| 빈 줄 필수 | 단일 줄바꿈은 무시 → 블록 사이 빈 줄. |
| 상류 의존 | 링크는 파일명 규칙, 요약은 일반 텍스트 컬럼에 의존. |
| UI 통제 한계 | 비동기 HITL은 본문만 통제 가능, 버튼·레이아웃 불가. |
