## 개념

적재 흐름이 만든 **이력서요약**은 AI가 쓴 것이라 품질을 사람이 한 번 확인해야 합니다. 그런데 담당자가 목록을 계속 들여다볼 수는 없으니, **새 지원자가 적재되는 사건**에 사람의 검토를 끼워 넣습니다. 이것이 **비동기 HITL**(Human-in-the-Loop)입니다.

| | 비동기 HITL (이 유닛) | 대화형 HITL (Unit 5) |
|---|---|---|
| 방식 | 시스템 이벤트에 승인 워크플로를 끼움 | 챗봇 대화 중 사람이 확인 |
| 트리거 | 항목 생성 (자동) | 면접관의 요청 (온디맨드) |
| 알림 | Teams·Outlook 자동 발송 | 챗봇 응답 |

이 승인이 곧 **게이트**입니다. 승인된 지원자만 하류(평가·면접)로 흘러가고, 승인 전에는 면접관 에이전트의 기본 조회에도 나타나지 않습니다. Unit 0에서 활성화한 **콘텐츠 승인**이 여기서 진가를 발휘합니다.

{: .note }
적재 흐름이 항목을 만들면, 콘텐츠 승인이 켜진 목록이라 그 항목은 자동으로 **보류 중** 상태가 됩니다. 승인 흐름은 이 보류 중 항목을 사람이 승인/거부해 **승인됨/거부됨**으로 바꾸는 일을 합니다.

---

## 단계별 가이드

### 1단계. 새 흐름 만들기

CS의 **흐름**에서 **`+ 새 흐름`** → 이름 **`승인 흐름`** 으로 자동화 흐름을 만듭니다.

![트리거 추가 패널]({{ site.baseurl }}/assets/unit2/u2-s01.png)  
*흐름 디자이너의 트리거 추가 패널. SharePoint·승인 관련 커넥터가 즐겨찾기·기본 제공 도구·커넥터별로 나열됩니다.*

### 2단계. 트리거 — "항목이 만들어지면"

SharePoint **`항목이 만들어지면`** 트리거를 선택하고 대상을 지정합니다.

| 항목 | 값 |
|---|---|
| 사이트 주소 | HR Lab |
| 목록 이름 | 지원자 마스터 |

![트리거 설정 완료]({{ site.baseurl }}/assets/unit2/u2-s02.png)  
*사이트 주소에 환경 변수 칩(HR_SPSiteURL), 목록 이름에 "지원자 마스터" 선택 완료.*

{: .note }
**"항목이 만들어지면"** 으로 둡니다. "만들어지거나 수정되면"으로 하면, 나중에 면접 확정 흐름(Unit 5)이 항목을 수정할 때마다 승인이 또 발송되는 부작용이 생깁니다.

### 3단계. 승인 시작 및 대기 액션 추가

트리거 다음에 **승인 — `승인 시작 및 대기`** 액션을 추가합니다.

| 항목 | 값 |
|---|---|
| 승인 유형 | `승인/거부 - 첫 번째로 응답` |
| 제목 | 예: `[승인요청] {지원자이름} 이력서 요약 검토` |
| 할당 대상 | 검토 담당자 메일 |

![승인 시작 및 대기 설정]({{ site.baseurl }}/assets/unit2/u2-s03.png)  
*제목(동적 칩 포함)·할당 대상·세부 정보(Markdown 카드)까지 완성된 상태. 세부 정보 작성 방법은 2-2에서 다룹니다.*

{: .important }
표준 **승인/거부** 형식을 그대로 씁니다. 버튼 라벨을 바꾸는 "사용자 지정 응답"도 있지만, 그러면 하류 조건 분기(`Approve`/`Reject` 비교)를 다시 배선해야 해서 실습에서는 표준을 유지합니다. "첫 번째로 응답"은 **할당자 여러 명 중 한 명**이 응답하면 종결된다는 뜻입니다.

{: .note }
이 액션 하나가 **Teams와 Outlook으로 승인 카드를 자동 발송**합니다. 별도 알림 액션을 만들지 않아도 됩니다. 카드의 내용(세부 정보)을 잘 쓰는 것이 다음 서브유닛의 주제입니다.

---

## ✅ 체크포인트

- [ ] CS에 **승인 흐름**이 생성되어 있습니다.
- [ ] 트리거가 **항목이 만들어지면**(지원자 마스터)으로 설정되어 있습니다.
- [ ] **승인 시작 및 대기**가 `승인/거부 - 첫 번째로 응답`으로 추가되어 있습니다.

---

## 핵심 정리

| 항목 | 내용 |
|---|---|
| 비동기 HITL | 사람 검토를 "항목 생성" 이벤트에 끼워 넣는 패턴. |
| 게이트 | 승인된 지원자만 하류로. 콘텐츠 승인이 그 장치. |
| 자동 알림 | 승인 액션 하나가 Teams·Outlook 카드를 자동 발송. |
| 트리거 선택 | "만들어지면"만 — "수정되면"은 재발송 부작용. |
