## 🎯 학습 목표

- 승인 흐름을 실행해 **Teams·Outlook 승인 카드 수신**을 확인한다.
- **승인 경로**와 **거부 경로**를 각각 실행해 결과 반영을 검증한다.
- 승인/거부에 따라 목록의 **승인상태**가 바뀌는지 확인한다.

## ⏱ 예상 소요 시간

{: .time }
약 10분

---

## 준비물

- 2-3까지 완료한 **승인 흐름**(저장·켜짐)
- Unit 1의 적재 흐름으로 **보류 중** 항목을 만들 수 있는 상태

---

## 단계별 가이드

### 1단계. 항목 생성으로 승인 트리거

적재 흐름을 통해(또는 목록에 직접 항목을 추가해) **새 지원자 항목**을 만듭니다. 콘텐츠 승인이 활성화되어 있어 항목은 **보류 중**으로 생성되고, 승인 흐름이 이를 감지해 실행됩니다.

<!-- SCREENSHOT: u2-s09 — 보류 중 상태로 생성된 새 항목 -->

### 2단계. Teams·Outlook 카드 수신 확인

담당자 계정의 **Teams**와 **Outlook**에 승인 카드가 도착했는지 확인합니다. 카드의 요약·링크가 2-2에서 만든 대로 정상 렌더링되는지 봅니다.

<!-- SCREENSHOT: u2-s10 — Teams/Outlook에 도착한 승인 카드 -->

{: .note }
한 번에 여러 지원자가 적재되면 카드도 여러 장 동시에 옵니다. 실제 강의에서는 6건이 한꺼번에 오는 양상을 함께 관찰합니다.

### 3단계. 승인 경로 검증

한 항목을 **승인**합니다. 흐름이 '예' 분기를 타고, 목록에서 해당 항목의 **승인상태가 승인됨**으로 바뀌는지 확인합니다.

<!-- SCREENSHOT: u2-s11 — 승인 후 승인상태 = 승인됨 -->

### 4단계. 거부 경로 검증

다른 항목을 **거부**합니다. 사유를 입력한 경우와 비워 둔 경우를 각각 시도해 봅니다.

- 승인상태가 **거부됨**으로 바뀌는가
- 지원자에게 회신 메일이 가는가
- 사유 입력 시 그 사유가, 빈 사유 시 **안내 문구로 폴백**되는가

<!-- SCREENSHOT: u2-s12 — 거부 회신 메일 (사유 폴백) -->

---

## ✅ 체크포인트

- [ ] 항목 생성 시 Teams·Outlook에 승인 카드가 도착합니다.
- [ ] 카드의 요약·링크가 정상 렌더링됩니다.
- [ ] 승인 시 승인상태가 **승인됨**으로 바뀝니다.
- [ ] 거부 시 승인상태가 **거부됨**으로 바뀌고, 사유(또는 폴백 문구)가 담긴 메일이 발송됩니다.

---

## 핵심 정리

| 항목 | 내용 |
|---|---|
| 자동 발송 | 승인 액션 하나로 Teams·Outlook 동시 알림. |
| 양 경로 검증 | 승인=승인됨, 거부=거부됨 + 사유 회신. |
| 게이트 완성 | 이제 승인된 지원자만 하류로. 다음 유닛의 에이전트가 이를 조회. |

{: .important }
이 유닛으로 **비동기 HITL** 한 패턴을 완성했습니다. Unit 5에서 만들 **대화형 HITL**과 합치면, "사람을 자동화의 어디에 어떻게 끼워 넣을 것인가"의 두 가지 답을 모두 갖추게 됩니다.
