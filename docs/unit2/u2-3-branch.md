## 🎯 학습 목표

- 승인 결과(승인/거부)에 따라 흐름을 **조건 분기**할 수 있다.
- **콘텐츠 승인 상태**를 흐름에서 승인됨/거부됨으로 설정할 수 있다.
- 거부 시 지원자에게 **사유를 담아 회신**하는 분기를 안전하게 구현한다.

## 개념

승인 액션이 끝나면 **결과(Outcome)** 값이 `Approve` 또는 `Reject`로 돌아옵니다. 이 값으로 흐름을 두 갈래로 나눕니다.

```
승인 결과?
├─ Approve → 콘텐츠 승인 상태 = 승인됨
└─ Reject  → 콘텐츠 승인 상태 = 거부됨 + 지원자에게 사유 회신 메일
```

핵심은 **승인 결과를 목록의 콘텐츠 승인 상태에 반영**하는 것입니다. 이래야 면접관 에이전트의 기본 조회(승인됨만)가 올바르게 동작합니다.

{: .important }
**승인상태는 직접 쓰는 컬럼이 아닙니다.** SharePoint 내장 콘텐츠 승인이라, 일반 "항목 업데이트"가 아니라 전용 액션(**콘텐츠 승인 상태 설정**)으로 바꿉니다. 챗봇·커넥터의 일반 업데이트로는 이 값을 못 바꿉니다(읽기 전용으로 거부됨) — 그래서 상태 변경은 흐름의 몫입니다.

---

## 단계별 가이드

### 1단계. 조건 분기 추가

승인 액션 다음에 **조건(Condition)** 을 추가하고, 승인 결과를 비교합니다.

![조건 노드 검색]({{ site.baseurl }}/assets/unit2/u2-s04.png)  
*작업 추가 패널에서 "조건"을 검색하면 컨트롤 섹션에 **조건** 노드가 나타납니다.*

조건 노드에서 비교할 값을 설정합니다. 동적 콘텐츠 패널에서 **승인 시작 및 대기** 섹션을 찾습니다. 항목이 많으면 **더 보기**를 눌러야 나타납니다.

![동적 콘텐츠 패널 — 승인 섹션]({{ site.baseurl }}/assets/unit2/u2-s05.png)  
*For each 1 → 승인 시작 및 대기 섹션 순으로 펼쳐집니다. "더 보기(25)"를 눌러야 전체 출력값이 나타납니다.*

![결과 칩 선택]({{ site.baseurl }}/assets/unit2/u2-s06.png)  
*Outcome은 한국어로 **"결과"** 로 번역됩니다. 목록에서 "결과 — 승인 결과"를 선택합니다.*

| 왼쪽 | 연산 | 오른쪽 |
|---|---|---|
| **결과** (= Outcome) | 다음과 같음 | `Approve` |

![조건 설정 완료]({{ site.baseurl }}/assets/unit2/u2-s08.png)  
*결과 칩 삽입 후 오른쪽에 `Approve`를 직접 입력합니다. 대소문자 정확히 일치해야 합니다.*

### 2단계. 예(True) — 승인됨으로 설정

**'예'** 분기에 SharePoint **`콘텐츠 승인 상태 설정`** 액션을 추가합니다. 노드 이름은 **승인 통과**로 변경합니다.

![True 분기 액션 추가]({{ site.baseurl }}/assets/unit2/u2-s09.png)  
*True 분기에서 "승인"으로 검색하면 SharePoint 섹션에 **콘텐츠 승인 상태 설정**이 나타납니다.*

| 항목 | 값 |
|---|---|
| 사이트 주소 | HR_SPSiteURL 환경 변수 칩 |
| 라이브러리 이름 | 지원자 마스터 |
| ID | 트리거의 **ID** 칩 |
| 작업 | `Approve` |

### 3단계. 아니요(False) — 거부됨 + 사유 회신

**'아니요'** 분기에 두 액션을 넣습니다.

먼저 **콘텐츠 승인 상태 설정** = `Reject`. 노드 이름은 **승인 거절**로 변경합니다.

이어서 **메일 보내기(V2)** 로 지원자에게 회신합니다.

![메일 보내기 — 동적 콘텐츠 사용 설정]({{ site.baseurl }}/assets/unit2/u2-s10.png)  
*받는 사람 필드 오른쪽의 ⚙️ 아이콘 → **동적 콘텐츠 사용**을 선택해야 칩을 삽입할 수 있습니다.*

![받는 사람 — 이메일 칩]({{ site.baseurl }}/assets/unit2/u2-s11.png)  
*동적 콘텐츠 패널에서 **항목이 만들어진 경우** 섹션의 **이메일** 칩을 선택합니다.*

**제목**에는 아래 텍스트를 붙여넣습니다.

```
지원 서류 확인 및 추가 보완 요청 안내
```

**본문**은 아래 텍스트를 먼저 붙여넣고, `{지원자이름}` 자리를 동적 칩으로 교체합니다.

```
안녕하세요, 채용담당자입니다.

저희 회사에 관심을 가지고 지원해 주셔서 진심으로 감사드립니다.

{지원자이름} 님의 보내주신 지원 서류를 1차 검토한 결과, 원활한 면접 진행을 위해 일부 서류(또는 정보)의 보완이 필요하여 안내해 드립니다.

아래 내용을 확인하신 후, 본 메일의 회신 또는 채용 페이지를 통해 서류를 보완하여 재제출해 주시면 감사하겠습니다.

■ 보완 필요 사항 (담당자 의견):
{사유}

감사합니다.
채용담당자 드림.
```

`{사유}` 자리에 거부 사유 식을 넣습니다. **응답 댓글** 칩을 먼저 삽입하면 For each로 자동 감싸지고, 그 식을 `coalesce`·`trim`·`if`로 감쌉니다.

![응답 댓글 칩 위치]({{ site.baseurl }}/assets/unit2/u2-s12.png)  
*comments는 한국어로 **"응답 댓글"** 로 번역됩니다. 승인 시작 및 대기 섹션 맨 아래에 있습니다.*

```
@{if(empty(trim(coalesce(items('For_each')?['comments'],''))),
   '담당자가 별도 의견을 남기지 않았습니다. 자세한 사항은 본 메일 회신으로 문의해 주시기 바랍니다.',
   items('For_each')?['comments'])}
```

{: .note }
`items('For_each')?['comments']` 를 직접 타이핑하지 말고, **동적 콘텐츠 → 승인 시작 및 대기 → 주석(Comments) 칩을 선택**하세요. 칩을 삽입하는 순간 PA가 자동으로 `For each` 루프로 감싸고 올바른 참조식을 생성합니다. 그 후 `coalesce`·`trim`·`if` 식으로 감싸면 됩니다.

![전체 완성 뷰]({{ site.baseurl }}/assets/unit2/u2-s13.png)  
*True(승인 통과)·False(승인 거절 + For each + 메일 보내기) 분기가 완성된 전체 구조.*

{: .warning }
거부 사유는 **승인 결과값(`approverResponse`)이 아니라 응답 댓글(`comments`)** 을 참조해야 합니다. 결과값을 꽂으면 사유를 입력해도 메일에 "Reject"가 찍힙니다(실제로 있었던 버그). 위 식은 `응답 댓글(comments)`을 참조하면서, 비어 있으면 안내 문구로 **폴백**합니다.

{: .note }
`coalesce`는 null 방어, `trim`은 공백만 입력한 경우 방어입니다. 표준 승인은 "거부 시 사유 필수"를 강제할 수 없어서, **흐름이 빈 사유를 방어**하는 설계입니다. (대화형 HITL은 반대로 지침으로 사유를 대화에서 강제 — 두 패턴의 대비 포인트.)

{: .note }
HTML 메일에서는 Markdown 볼드(`**이름**`)가 먹지 않아 별표가 그대로 노출됩니다. 메일 본문에서는 별표를 빼고 작성하세요(승인 카드와 규칙이 다름).

## ✅ 체크포인트

- [ ] 승인 결과로 **예/아니요 분기**가 갈립니다.
- [ ] 예 분기: 콘텐츠 승인 상태가 **승인됨**으로 설정됩니다.
- [ ] 아니요 분기: **거부됨** 설정 + 사유 회신 메일이 발송됩니다.
- [ ] 거부 메일 사유가 `comments` 참조 + 빈값 폴백으로 작성되어 있습니다.

---

## 핵심 정리

| 항목 | 내용 |
|---|---|
| 결과 분기 | Outcome(한국어: **결과**) `Approve`/`Reject`로 두 갈래. |
| 승인 통과·거절 노드 | "콘텐츠 승인 상태 설정" 액션 이름을 변경해 가독성 확보. 승인상태는 전용 액션으로만 변경 가능(일반 업데이트 불가). |
| 사유 폴백 | **응답 댓글**(comments) 참조 + 빈값 시 안내 문구. `approverResponse` 아님. |
| 방어적 설계 | 비동기 HITL은 UI를 못 막으니 흐름이 빈 사유를 방어. |
