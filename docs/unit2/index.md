# Unit 2. 승인 흐름 ✅

AI가 만든 요약, 그대로 믿어도 될까요? 이 유닛에서는 **비동기 HITL**(Human-in-the-Loop) — 사람의 검토를 시스템 이벤트에 끼워 넣는 패턴을 만듭니다. 지원자 항목이 생성되는 순간 담당자에게 Teams·Outlook 승인 카드가 자동 발송되고, 승인 전까지 그 지원자는 에이전트의 기본 조회에 나타나지 않습니다.

핵심은 "AI 요약의 품질을 사람이 보증한 데이터만 하류(평가·면접)로 흘려보낸다"는 **게이트 설계**입니다. Unit 0에서 켜 둔 콘텐츠 승인이 여기서 진가를 발휘합니다.

## 이 유닛을 마치면

- 항목 생성 → 승인 요청 발송 → 승인/거부 결과 반영까지 자동화됩니다.
- 승인 카드에 요약·이력서 링크를 보기 좋게 싣는 방법(Markdown 규칙)을 익힙니다.
- 거부 시 지원자에게 사유를 담아 회신하는 분기 처리를 구현합니다.

{: .note }
승인 카드 세부 정보는 **Markdown 전용**입니다. HTML 태그는 해석되지 않고, 빈 줄 규칙을 지키지 않으면 서식이 무너집니다 — 본문에서 함정 포인트로 다룹니다.
