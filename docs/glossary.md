# 부록. 용어집 📚

이 과정에 등장하는 약어와 핵심 용어를 모았습니다. 본문을 읽다가 낯선 말이 나오면 여기로 돌아오세요.

## 약어

| 약어 | 풀네임 | 한 줄 설명 |
|---|---|---|
| **SP** | SharePoint | Microsoft 365의 협업·데이터 플랫폼. 이 과정에서 지원자 명단(목록)과 이력서 파일(문서 라이브러리)의 집 |
| **HITL** | Human-in-the-Loop | AI 자동화 중간에 **사람의 검토·승인·결정**을 끼워 넣는 설계 패턴. 이 과정의 핵심 주제 |
| **PAD** | Power Automate Desktop | PC 화면을 직접 조작하는 데스크톱 자동화(RPA) 도구. 오프닝 시연에 등장 |
| **CS** | Copilot Studio | Microsoft의 에이전트(챗봇) 제작 플랫폼. 이 과정의 주 무대 |
| **RAG** | Retrieval-Augmented Generation | 검색 증강 생성 — 질문과 관련된 문서 조각을 찾아 AI 답변의 근거로 쓰는 기술. Knowledge가 이 방식으로 동작 |
| **JD** | Job Description | 직무기술서 — 직군별 필수/우대 요건을 정의한 문서. Knowledge 4종 중 하나 |

## 핵심 개념

| 용어 | 설명 |
|---|---|
| **목록 (List)** | 행·열로 된 SharePoint 데이터 저장소. 지원자 한 명 = 한 항목(행), 속성 = 컬럼. 이 과정의 단일 진실 원천 (Unit 0) |
| **내부 이름 (Internal Name)** | 컬럼을 처음 만든 이름으로 고정되는 시스템 식별자. 흐름·식이 이 이름으로 컬럼을 참조 — 영문으로 만들어야 다루기 쉬움 (Unit 0) |
| **흐름 (Flow)** | 트리거 → 액션 순서로 실행되는 자동화 단위. 항상 같은 입력에 같은 동작을 하는 **결정적** 실행 |
| **커넥터 (Connector)** | 에이전트·흐름이 외부 시스템(SharePoint, Outlook 등)과 대화하는 연결 통로 |
| **에이전트 (Agent)** | Copilot Studio로 만드는 AI 챗봇. Knowledge·도구·지침을 조합해 동작 |
| **Knowledge** | 에이전트에 등록하는 참조 문서. "기준"을 담는 곳 (이력서 같은 "대상 데이터"는 넣지 않음 — Unit 3) |
| **코퍼스 (Corpus)** | 검색 대상이 되는 문서 전체 더미. RAG는 이 더미 전체에서 관련 조각을 찾는다. 이력서 수백 개를 한 더미에 넣으면 "이 한 명"만 깔끔히 보기 어려운 이유 (Unit 3·4) |
| **지침 (Instructions)** | 에이전트의 행동 규칙을 자연어로 쓴 것. 시스템 프롬프트에 해당 |
| **가드레일 (Guardrail)** | 지침으로 에이전트의 행동 범위를 좁히는 제약. 도로 난간처럼 "이 도구만", "승인된 사람만", "없는 건 지어내지 마"로 길 밖으로 못 나가게 막는다. 줄일 수 있는 모호함을 줄여 오답 범위를 좁히는 것 (Unit 4·5) |
| **토픽 (Topic)** | 에이전트 안에서 특정 주제의 대화를 처리하는 단위 시나리오 |
| **콘텐츠 승인 / 승인상태** | SharePoint 내장 기능. 항목을 승인됨/거부됨/보류 중으로 관리 — 직접 만드는 컬럼이 아님 (Unit 0·2) |
| **검토상태** | 채용 진행 단계를 담는 직접 만든 컬럼 (검토중/1차합격/보류/불합격) — 승인상태와 별개 (Unit 5) |
| **트랜잭션** | 여러 데이터 변경이 "전부 성공 아니면 전부 취소"로 묶여야 하는 작업. 흐름이 필요한 결정적 이유 (Unit 5) |
| **평가 루브릭** | 지원자 적합도를 등급으로 환산하는 채점 규칙 문서. Knowledge 4종의 키스톤 (Unit 3·4) |
| **시티즌 개발자** | 전문 개발자가 아니지만 로우코드 도구로 업무 자동화를 직접 만드는 현업 담당자 — 바로 여러분 |
| **AI Builder** | Power Automate 안에서 쓰는 AI 기능 모음. 적재 흐름의 이력서 정보 추출에 사용 (Unit 1) |
| **C1 / C2 / C3 (모듈 1 기능)** | 면접관 에이전트가 **커넥터 도구 하나 + 지침**으로 제공하는 세 기능 — C1 지원자 조회, C2 적합도 평가, C3 면접 질문 생성. 서로 다른 흐름이 아니라 한 도구가 지침으로 갈라지는 의도다 (Unit 4). 상태를 바꾸는 면접 확정만 흐름(Unit 5) |
