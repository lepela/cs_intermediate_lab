# Unit 0. 환경 셋업 🏗

자동화의 첫 단추는 코드도 흐름도 아닌 **데이터가 살 집**입니다. 이 유닛에서는 채용 데이터의 집이 될 SharePoint 환경을 만듭니다 — 사이트, 지원자 마스터 리스트, 이력서 원본을 보관할 문서 라이브러리, 그리고 사람의 승인을 시스템에 끼워 넣는 장치인 **콘텐츠 승인**까지.

여기서 만드는 리스트 컬럼 하나하나(승인상태·검토상태·이력서요약·이력서링크)는 이후 모든 유닛이 그대로 사용합니다. 컬럼 설계가 곧 전체 솔루션의 뼈대인 셈입니다.

## 이 유닛을 마치면

- HR Lab 커뮤니케이션 사이트가 생성되어 있습니다.
- 지원자 마스터 리스트 + 필수 컬럼 스키마가 완성됩니다.
- 콘텐츠 승인이 활성화되어 "사람의 승인 없이는 공개되지 않는" 데이터 구조가 준비됩니다.

{: .important }
**승인상태**는 직접 만드는 컬럼이 아니라 SharePoint **내장 콘텐츠 승인** 기능입니다. 이 차이가 Unit 2(승인 흐름)와 Unit 5(트랜잭션)의 핵심 복선이 됩니다.
