## 🎯 학습 목표

- Power Platform **솔루션**이 무엇인지, 왜 솔루션 안에서 작업해야 하는지 이해한다.
- 솔루션에 **환경 변수**를 등록하고, 하드코딩 대신 변수를 쓰는 이유를 이해한다.
- 이후 유닛에서 흐름과 에이전트가 참조할 환경 변수를 미리 만들어 둔다.

## 개념

### 솔루션 — 컴포넌트를 담는 컨테이너

흐름·에이전트·환경 변수를 **솔루션** 안에서 만들면 세 가지를 얻습니다.

| 이점 | 설명 |
|---|---|
| 이식성 | 솔루션 하나를 내보내면(export) 흐름·에이전트·변수가 한 묶음으로 이동 |
| 환경 분리 | 개발 → 운영 환경 이동 시 환경 변수 값만 교체하면 됨 |
| 관리 편의 | 이 프로젝트 것만 한 곳에 모여 있어 찾기 쉬움 |

{: .note }
솔루션 밖에서 만든 흐름도 동작하지만, 나중에 솔루션으로 옮기거나 내보내기 어렵습니다. **처음부터 솔루션 안에서 만드는 습관**이 중요합니다.

### 환경 변수 — 하드코딩 대신 참조

흐름 안에 SharePoint 사이트 URL을 직접 입력하면, 사이트가 바뀌거나 환경이 달라질 때 흐름을 열어 일일이 수정해야 합니다. **환경 변수**에 값을 한 번 등록해 두면, 흐름·에이전트 어디서든 그 변수를 참조하고, 값 변경은 변수 한 곳에서만 하면 됩니다.

| 등록할 환경 변수 | 타입 | 예시 값 |
|---|---|---|
| `HR_SPSiteURL` | 텍스트 | `https://tenant.sharepoint.com/sites/hr-edu` |

## 단계별 가이드

### 1단계. Power Apps 메이커 포털에서 새 솔루션 만들기

[make.powerapps.com](https://make.powerapps.com) 에 접속 → 왼쪽 메뉴 **솔루션(Solutions)** → **`+ 새 솔루션`** 을 클릭합니다.

<!-- SCREENSHOT: u0-s24 — 솔루션 목록에서 '+ 새 솔루션' 클릭 -->

아래와 같이 입력합니다.

| 필드 | 값 |
|---|---|
| 표시 이름 | `HR 채용 자동화` |
| 이름(내부) | `HRRecruitment` (자동 생성, 그대로 사용) |
| 게시자 | 기본 게시자 선택 |
| 버전 | `1.0.0.0` |

<!-- SCREENSHOT: u0-s25 — 새 솔루션 만들기 폼, 표시 이름·이름 입력 -->

**만들기**를 클릭하면 솔루션이 생성되고 솔루션 내부 화면으로 이동합니다.

### 2단계. 솔루션 안에서 환경 변수 추가

솔루션 내부 화면 상단 **`+ 새로 만들기`** → **환경 변수(Environment variable)** 를 선택합니다.

<!-- SCREENSHOT: u0-s26 — 솔루션 내부 '+ 새로 만들기 > 환경 변수' 메뉴 -->

아래 값을 입력하고 **저장**합니다.

**HR_SPSiteURL**

| 필드 | 값 |
|---|---|
| 표시 이름 | `HR_SPSiteURL` |
| 이름 | `hr_SPSiteURL` (자동) |
| 데이터 형식 | 텍스트 |
| 현재 값 | `https://<테넌트>.sharepoint.com/sites/hr-edu` |

<!-- SCREENSHOT: u0-s27 — HR_SPSiteURL 환경 변수 등록 완료된 솔루션 내부 목록 -->

{: .important }
**현재 값(Current value)** 과 **기본값(Default value)** 은 다른 필드입니다. 이 실습에서는 **현재 값**에 입력합니다. 현재 값이 없으면 기본값을 참조하므로 어느 쪽에 넣어도 동작하지만, 환경마다 달라지는 값은 현재 값에 두는 것이 관례입니다.

## ✅ 체크포인트

- [ ] `HR 채용 자동화` 솔루션이 메이커 포털에 생성되어 있습니다.
- [ ] 솔루션 안에 환경 변수 `HR_SPSiteURL`이 등록되어 있습니다.
- [ ] 각 변수의 **현재 값**이 실제 환경에 맞게 입력되어 있습니다.

## 핵심 정리

| 항목 | 내용 |
|---|---|
| 솔루션 | 흐름·에이전트·변수를 한 묶음으로 관리하는 컨테이너. 처음부터 솔루션 안에서 만든다. |
| 환경 변수 | URL·이름 등 환경마다 달라지는 값을 한 곳에 등록. 흐름·에이전트는 값이 아닌 변수를 참조. |
| 이식성 | 솔루션 export → import 한 번으로 다른 환경에 동일한 자동화를 배포할 수 있다. |
